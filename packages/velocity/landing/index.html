Your coding agent's speedometer

velocity-mcp gives AI coding agents self-awareness of their own execution speed. Time every task. Estimate every plan. Know how long things actually take.

$ npx velocity-mcp Click to copy
The problem
Coding agents are time-blind

Claude Code, Cursor, Codex — they can plan, code, debug, and ship. But ask them "how long will this take?" and they have nothing. No clock. No history. No self-awareness of their own speed.

⏱

No ETAs on plans

Your agent creates a 12-step implementation plan. You have no idea if that's 15 minutes or 2 hours of agent execution time.

🚫

Data evaporates

Every completed task is a data point about agent throughput. But it vanishes after each session — zero learning, zero memory.

🔍

No performance visibility

Is your agent slower at debugging than scaffolding? Is TypeScript faster than Python? You'll never know without measurement.

📅

Multi-session plans are timeless

Plans that span days or weeks have no temporal structure. "Phase 2" means nothing without knowing how long Phase 1 took.

How it works
Three lines in your CLAUDE.md

Install the MCP server. Tell your agent to use it. Every task gets timed, categorized, and stored. Over time, your agent learns its own speed.

1

Install

One command. Works with any MCP-compatible client.

claude mcp add velocity-mcp -- npx velocity-mcp
2

Instruct

Add a few lines to your CLAUDE.md telling the agent to call velocity_start_task before each task and velocity_end_task after. That's it.

3

Accumulate

Every completed task builds your agent's velocity profile in a local SQLite database. After a few sessions, estimation accuracy improves dramatically.

4

Estimate

Ask your agent to estimate a plan. It queries its own historical data and gives you a real, data-backed ETA — not a guess.

"This 8-step plan should take approximately 22 minutes based on 47 similar tasks."
MCP Tools
Five tools. Zero config.
velocity_start_task
Begin timing a task with category, tags, and metadata. The agent calls this before starting any discrete unit of work.
// Agent starting a new component { "category": "implement", "tags": ["react", "typescript"], "description": "Login form component" }
velocity_end_task
Stop the timer and record the result. Includes a comparison against historical average for the same task type.
// Returns performance context { "duration": "3m 7s", "vs_avg": "-26% faster", "category": "implement" }
velocity_estimate
Pass in a plan (list of subtasks) and get a total time estimate with per-task breakdown and confidence levels.
// Estimate a 4-step plan { "total": "18m 30s", "confidence": "medium", "tasks": 4, "based_on": 23 similar tasks }
velocity_stats
Query aggregate performance data grouped by category, tag, project, or time period. Your agent's performance dashboard.
// Last 30 days breakdown implement avg 4m 12s × 22 tasks debug avg 6m 30s × 11 tasks test avg 5m 00s × 8 tasks
velocity_history
Review recent task records with full metadata. Filter by category, status, or project. Useful for debugging slow tasks or reviewing what the agent actually did in previous sessions.
