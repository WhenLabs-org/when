The toolkit
Everything your agent needs

Six purpose-built tools for modern developer workflows. Each one does one thing well.

velocity
MCP Server
Task timing and estimation for coding agents. Builds a local history so your agent knows its own speed.
$ when velocity stats
stale
CLI
Detect documentation drift in your codebase. Flags docs that haven't kept up with code changes.
$ when stale check
envalid
CLI
Validate .env files against a type-safe schema. Catch missing or malformed environment variables before they hit prod.
$ when envalid check
berth
CLI
Detect and resolve port conflicts. Find what's running on a port and free it up — no more EADDRINUSE surprises.
$ when berth scan
aware
CLI
Auto-detect your stack and generate AI context files. Gives your coding agent a map of your project from day one.
$ when aware scan
vow
CLI
Scan dependency licenses and validate against policies. Enforce allowlists before a non-compliant dep sneaks into prod.
$ when vow scan
How it works
Install once. Use everywhere.

One package installs six tools. Each tool works standalone from the CLI. velocity also wires up as an MCP server so your coding agent gets real-time task timing and historical estimates.

1

Install

Run one command to install the full toolkit. All six tools are available immediately as when <tool> subcommands.

2

Use

Run tools from your terminal or let your coding agent call them. velocity hooks into Claude Code or any MCP-compatible client to time every discrete task your agent performs.

3

Improve

Every timed task builds your velocity profile in a local SQLite database. Your agent uses this history to give data-backed ETAs on future plans — the flywheel compounds over sessions.
